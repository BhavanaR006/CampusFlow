## 📋 Technical Setup (Before Recording)

### Screen Recording with QuickTime (Mac)

1. **Open QuickTime Player** → File → New Screen Recording
2. **Click the dropdown arrow** next to the record button:
   - Microphone: **Built-in Microphone** (ON)
   - System Audio: **OFF** (no clicks/notifications)
   - Show Mouse Clicks: **ON** (helps viewers follow)
3. **Set resolution:** System Preferences → Displays → Set to **1920×1080** (scaled)
4. **Preparation:**
   - Close all other apps (no notifications)
   - Open Chrome/Safari in full screen
   - Navigate to: `http://localhost:8000` (or your S3 URL)
   - Clear browser cache for clean demo
   - Make sure terminal is running: `cd backend && python main.py`

### Export Settings

1. After recording: File → Export As → **1080p**
2. Format: `.mov` (QuickTime native)
3. Convert to MP4 (if needed): `ffmpeg -i demo.mov -c:v libx264 -crf 23 -preset medium demo.mp4`
4. Target: Under **100MB** for upload (3.5 min at 1080p ≈ 50-80MB)

### Browser Setup

- **Browser:** Chrome (Dark mode matches the app)
- **Window size:** Full screen, 1920×1080
- **Zoom:** 100% (no scaling)
- **Bookmarks bar:** Hidden (Cmd+Shift+B)
- **Tab:** Only CampusFlow open

## 🏆 Pro Tips for High Scores

1. **Speak with confidence** — you built this, own it
2. **Pause on data** — let the jury read the attendance numbers, budget figures
3. **Mouse movements are slow and deliberate** — no frantic clicking
4. **The AI responses are the wow moment** — pause after "Can I skip CN?" to let the response load dramatically
5. **End strong** — the closing line should feel like a mic drop
6. **Keep it under 3:30** — respect the jury's time, shows discipline
